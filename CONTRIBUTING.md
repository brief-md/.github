# Contributing to BRIEF.md

Thanks for your interest in contributing. We're early and the ecosystem is wide open — there's room for all kinds of contributions.

---

## Ways to Contribute

**Improve the spec**
If you spot something unclear, incorrect, or missing in the specification, open an issue or submit a pull request in the [spec repo](https://github.com/brief-md/spec).

**Build a tool or integration**
If you're building something on top of BRIEF.md — an integration, a parser, a type guide, a domain extension, a plugin — open an issue to let us know. We'd rather know about it early than after the fact.

**Write a type guide**
Type guides define how BRIEF.md works in a specific domain. If you have domain expertise and want to define how BRIEF.md should work for your field, that's a valuable contribution. Start by opening an issue with a rough outline.

**Report a bug**
Use the bug report template in the relevant repo. The more specific the better.

**Propose something new**
Use the feature request template. Explain the problem you're trying to solve, not just the solution you have in mind.

---

## Before You Start

For anything beyond a small fix, open an issue first and describe what you're planning. This avoids duplicated effort and makes sure the contribution fits the direction of the project before you invest time in it.

---

## How to Submit

1. Fork the relevant repo
2. Create a branch with a descriptive name
3. Make your changes
4. Open a pull request with a clear description of what you changed and why

---

## What We're Looking For Right Now

- Feedback on the spec from people using it in real projects
- Type guides for domains beyond music
- Integrations with MCP-compatible tools
- Parsers and validators in languages beyond the reference implementation
- Real world examples of BRIEF.md in use

## Integrating BRIEF.md Into Your Tool or Service

If you're building a tool or service and want to add BRIEF.md support, we want to hear from you early. That could mean reading BRIEF.md files to load project context, writing back to them during a session, supporting the hierarchy model, or building on top of the MCP server.

There's no formal integration programme yet. Get in touch at [hello.briefmd@gmail.com](mailto:hello.briefmd@gmail.com) and we can work out what support looks like.

---

## AI-Assisted Contributions

AI-assisted contributions are welcome. What matters is that you have understood and tested what you are submitting. The quality bar is the same regardless of how it was written.
